# 一、案例分析

核心提示：案例分析5选3，架构设计为高频必考大题，答题核心=架构选型+三高方案+风险解决，以下知识点直接对应考题考点，优先背诵标粗内容。

## 一、核心架构风格（必考，选型题/分析题高频）

答题关键：先判断业务场景，再选架构，说明选型理由（适配性+优势）

- **分层架构（三层/四层）**
  - 核心结构：表现层（UI）→ 业务逻辑层（服务）→ 数据访问层（DAO）→ 数据层（DB）
  - 优势：结构清晰、解耦简单、开发维护成本低、适配中小型系统
  - 适用场景：业务简单、体量小、团队规模不大、需求稳定（如内部管理系统）
- **微服务架构**
  - 核心结构：按业务垂直拆分（如用户服务、订单服务），独立开发、独立部署、独立数据库（去中心化）
  - 优势：高可扩展、容错性强、多团队并行开发、适配高并发大项目
  - 适用场景：业务复杂、模块独立、高并发、高扩展需求（如电商、支付系统）
  - 核心组件：服务注册发现（Nacos/Eureka）、网关（Gateway）、配置中心、链路追踪
- **事件驱动架构**
  - 核心结构：基于消息事件（MQ）实现异步通信，生产者→消息队列→消费者，无直接依赖
  - 优势：解耦性极强、高吞吐、削峰填谷、适配实时业务
  - 适用场景：实时消息推送、日志处理、异步任务（如订单通知、消息推送）
  - 常用组件：RabbitMQ、Kafka
- **CQRS架构**
  - 核心结构：命令（写操作）与查询（读操作）职责分离，分开部署、分开存储
  - 优势：提升查询性能、读写互不影响、适配查询量大的场景
  - 适用场景：查询量大、写入少（如报表系统、日志查询系统）
- **管道-过滤器架构**
  - 核心结构：数据流式处理，管道传递数据，过滤器负责数据处理（如解析、转换）
  - 适用场景：日志分析、数据清洗、流式计算（如大数据处理场景）

## 二、架构设计核心原则（必背，答题必写补充）

- 基础原则：**SOLID五大原则**（单一职责、开闭原则、依赖倒置、接口隔离、迪米特法则）
- 核心原则：**高内聚、低耦合**（模块内部紧密，模块间依赖弱）
- 延伸原则：可扩展性、可维护性、安全性、复用性、容错性

## 三、三高架构方案（核心考点，案例分析必答）

答题关键：无论哪种架构，都需补充“三高”实现方案，贴合业务场景即可

### 1. 高可用（避免单点故障，保证系统持续运行）

- 核心方案（必背）：集群部署、主从切换（MySQL主从）、故障转移
- 补充方案：服务熔断（Sentinel）、服务降级、限流、超时控制、重试机制
- 高级方案：异地多活、容灾备份（本地+异地备份）、故障自愈

### 2. 高性能（提升系统响应速度）

- 缓存优化（必背）：本地缓存（Caffeine）、分布式缓存（Redis）、多级缓存
- 异步处理：线程池、MQ异步通信（削峰填谷）
- 数据库优化：索引优化、读写分离、分库分表
- 接口优化：接口合并、懒加载、数据压缩传输、负载均衡（Nginx/LVS）

### 3. 高可扩展（应对业务增长，灵活扩容）

- 核心方案：微服务拆分、服务无状态化、水平扩容（增加节点）
- 补充方案：配置中心（动态配置）、插件化架构、模块化设计

## 四、分布式架构核心问题（案例分析高频问答）

- **分布式事务**：解决数据一致性，必背方案（按优先级）：TCC、SAGA、本地消息表、可靠消息最终一致性（避免死记2PC，缺陷多）
- **分布式锁**：解决并发冲突，常用方案：Redis分布式锁、ZooKeeper分布式锁
- **服务调用问题**：服务注册发现、负载均衡、接口幂等性（解决重复请求，方案：唯一标识、幂等设计）
- **链路追踪**：排查分布式问题，常用组件：SkyWalking、Zipkin

## 五、架构风险&解决方案（必背，案例分析压轴题）

| 核心风险                   | 解决方案（必写）                                             |
| :------------------------- | :----------------------------------------------------------- |
| 单点故障                   | 集群冗余、主从切换、异地多活                                 |
| 服务雪崩                   | 服务熔断、降级、限流、超时控制                               |
| 数据一致性                 | 分布式事务（TCC/SAGA）、最终一致性设计                       |
| 调用混乱                   | 网关统一收口、服务注册发现、接口规范                         |
| 缓存异常（穿透/击穿/雪崩） | 穿透：布隆过滤器+空值缓存；击穿：互斥锁+热点永不过期；雪崩：过期时间打散+集群 |



## 六、案例答题万能模板（直接套用）

1. 选型：根据题干业务场景，确定核心架构风格（如“业务复杂、高并发，选用微服务架构”）
2. 架构设计：画出核心架构图（简要描述分层/模块拆分），说明各模块职责
3. 三高实现：结合选型，补充高可用、高性能、高可扩展方案（贴合业务，不堆砌）
4. 风险防控：指出可能存在的风险（如微服务调用混乱），给出对应解决方案
5. 总结：说明选型优势，适配业务需求，保证系统稳定、可扩展

重点提醒：案例分析不考死记硬背，需结合题干业务场景灵活套用知识点，优先写标粗核心内容，再补充延伸内容，保证答题逻辑清晰、踩点准确。
