## 第一章：系统架构概述&常见分类与建模方法

### 1.1、系统架构概述

**概念：**架构是组件中一个系统的级别组织，系统是组织完成某一功能的组件集。

**系统架构：** 是高层次的结构表示是系统骨架和根基，包括（**组件、连接件、约束规范**）指导这些内容的设计和演化原理如：需求分析->系统设计->实施->测试->验收->交互；需求与设计之间高层次宏观的系统架构设计。

**架构设计作用包括以下几点：**

1. 解决相对复杂的需求分析；

2. **解决非功能属性（性能、安全性、可用性、可靠性）在系统占据中重要位置的设计问题；**

3. 解决生命周期长、拓展性需求高的系统整体结构问题；

4. 解决系统基于组件需求的集成问题；

5. 解决业务流程组件的集成问题；

<u>**系统架构设计：**是成熟系统开发的重要节点，需求和系统进一步设计与实现的桥梁，是系统早期阶段质量的关键步</u>骤。

### 1.2、软件架构的发展阶段 *

**模块化开发方法：**模块设计规则，系统如何分解成模块，每个模块都可以独立开发与测试、最后组装成一个完整的软件，规则如下：

1. <u>**高模块内聚合**</u>
2. <u>**最低耦合**</u>
3. 模块大小合适
4. 模块调用链的深度不可过多
5. 接口简单，精练（扇入扇出不太大）具有信息隐蔽能力
6. 尽可能的复用已有模块

**组件化技术：**具有可组装性和可插拔性，各组件运行依赖平台或容器，组件与组件间不存在直接耦合性，且组件与组件之间非绝对独立，和其他组件有业务上交互。**组件化比模块化更加独立，但比集成结合更加紧密。**

**软件架构描述与表示：**软件架构分析、设计与测试、软件架构发现、演化与重用。

**软件架构开发方法：**软件架构风格、动态软件架构等。

**软件架构：**是软件生命周期中重要的产物，他影响软件开发的各个阶段。

### 1.3、软件架构的常用分类及建模方法

<u>**经典架构模型：**分层架构、事件驱动架构、微核架构、微服务架构、云架构等 5 类</u>

**分层架构：**表现层（用户界面、负责视觉与互动）、业务层（实现业务逻辑）、持久层（提供数据）、数据库（存储数据）

在逻辑层与持久层之间加一服务层实现通用接口和通用逻辑用户依次通过四层。

**事件驱动架构EDA：**<u>通过事件进行通信，是一种流行的分布式一部架构模式，适用于松散耦合，具体如下</u>：

1. **事件队列：**接受事件的入口
2. **分发器：**将不同得事件分发到不同的业务逻辑单元
3. **事件通道：**分发器和处理器之间的联系
4. **事件处理器：**实现业务逻辑，处理完成后发出事件触发下一步操作。

**微内核架构：**又称插件架构、功能单一，功能最小，如：模块加载，通讯功能。如有拓展通过插件实现业务逻辑，相互依赖。

**微服务架构SOA：**<u>是服务导向架构的升级，每个服务都是独立的单元部署。</u>

1. **RESTful API ：**通过API 提供云服务
2. **RESTful 应用模式：** 服务通过传统的网络协议或者应用协议，通常是应用程序
3. **集中消息模式：**采用消息代理，实现消息队列，负载均衡，统一处理异常，缺点会出现单点失败，消息代理需要做成集群。

**云架构：**主要解决拓展性和并发问题，是最容易拓展得架构，是高拓展性现在将数据都复制到内存中，可复制得内存数据单元，然后将业务处理能力封装成一个处理单元。

1. **处理单元：**实现业务逻辑
2. **虚拟中间件：**负责通信、保持会话控制、数据复制、分布式处理和处理单元的部署。
3. **消息中IIE件：**管理用户请求和会话控制
4. **数据中间件：**将数据复制到每一个处理单元，即数据同步、保证某处理单元都得到同样的数据。
5. **处理中间件：**请求涉及到不同的类型的处理单元
6. **部署中间件：**负责处理单元启动，监控负载和响应时间。

### 1.4、软件架构模型（重点）：

**系统架构的建模方法：**结构模型、框架模型、动态模型、过程模型

1. **结构模型：**最直观、普遍的建模方法。用构件、连接件或其他概念来刻画结构。核心是架构描述语言。
2. **框架模型：**与结构模型类似，侧重整体结构。
3. **动态模型：**研究（大颗粒）行为的性质。
4. **过程模型：**步骤和过程。

以上四种模型并不是完全独立的，通过有机结合形成完整的模型来刻画软件架构。

### 1.5、软件架构和应用场景和未来发展（重点）

**架构风格：**

1. **管道-过滤器风格：**面向数据流的风格，用于将系统分成若干独立的步骤。
2. **主程序/子系统和面向对象的架构风格：**用于组件内部进行设计。
3. **虚拟机风格：**用于构造解释器和专家系统。
4. **C/S 和B/S风格：**适用于数据和处理分布在一定方位，通过网络链接构成系统。
5. **平台/插件风格：**插件拓展功能的应用程序。
6. **MVC风格：**用户交互程序的设计。
7. **SOA风格：**用在企业集成等方面。
8. **C2风格：**用于GUI软件开发，用于构建灵活的可拓展的应用系统等（构件，连接件）。
9. **数据仓储：**集成开发环境采用

**软件架构未来发张：**主线模块化编程、面向对象编程、构件技术、面向服务开发技术和云技术。

**总结：** 五架构、四模型、九风格

## 第二章：计算机基础知识

### 2.1、硬件概述

- **计算机系统：**数据管理、软件、硬件、组成测网络系统，他是要求人接收和存储信息，自动进行数据处理和计算，并并输出结果的机器系统。
- **硬件：**
  1. 中央处理器CPU（运算单元、控制单元）程序计数器、指令寄存器（当前指令）

**计算机硬件5部分：**输入设备、输出设备、存储器、运算器、控制器

1. **处理器（控制器和运算器）：**核心部件、复杂指令集（CISC）、精简指令（RISC）、信号处理器（DSP）用于实时信号数据处理

2. **存储器（重点）：**SRAM、DRAM、NVRAM、Flash、EPROM、Disk等。层次具体区分如下：

   ​	**片上缓存：**CPU 中 的缓存，SRAM结构，容量小，速度快。

   ​	**片外缓存：**CPU 外 的缓存，将主存的部分常用数据引入程序，存储Cashe，CPU 直接交互。

   ​	**主存：**RAM

   ​	**外存：**ROM

3. **总线（##）：**<u>分：内总线、系统总线、外部总线，具</u>体如下：

   **内总线：**用于各类芯片的内部链接

   **系统总线：**是指计算机中的CPU、主存、I/O接口的总线

4. **接口：**不同层之间的通讯规则

5. **外部设备：**输出设备、输出设备

### **2.2、软件概述**

- **软件系统（#）：**是硬件系统上的程序，相关文档资料的数据集合，<u>通常将计算机软件分为，系统软件、应用软件两大类。</u>

  1. **系统软件：**整个计算机系统配置的不依赖特定应用领域的通用软件。
  2. **应用软件：**解决某个特定需求而设计的软件。

  **重点：**<u>计算机硬件（操作系统开发人员）->操作系统->系统软件（应用开发人员）->应用软件（应用用户）</u>

- **操作系统特征（###）：**

  1. **并发性：**本质分时，宏观上多个程序同时运行，每一时刻只有一个程序在运行。
  2. **共享性：**资源被多个并发执行的进程共同使用，而不是被一个经常独占。
  3. **虚拟性：**物理实体变成逻辑上的多个对应物（如ROM分区），物理上多个实体，变成逻辑上的对应物技术（双通道内存16*16GB 变成32GB）
  4. **不确定性：**多程序运行中，允许多个进程并发运行，走走停停

- **硬件与软件操作系统分类（##）：**

  1. **批处理系统：**单道批处理，用户可以提交多个作业，一次执行一次，并行；多道批处理，一次允许多个作业封入内存执行，在某时刻，作业都一起开始或结束；特点：多道、宏观三并行运行和微观上串行运行。
  2. **分时操作系统：**<u>以时间分片，轮流使用，多路性、独立性、交互性、即时性。</u>
  3. **实时操作系统：**<u>处理速度快，被控对象允许时间范围内做出的快速反应。</u>
  4. **嵌入式操作系统：**微型化、可定制、实时性、可靠性、易移植性。

### 2.3、文件存储管理

- **直接索引：**索引指向物理盘块

- **一级间接索引：**索引指向索引

- **二级简介索引：**索引指向索引表在指向物理盘块

  

### 2.4、文件系统（重点）

- **文件存储方法：**顺序存储与随机存储两种方法

- **文件存储空间管理：**空闲区表、位示图、空闲块链。

  1. **空闲区表：**外存空间上连续未分配的区域称之为空闲区表。
  2. **位示图：**外存建立该图，每一位对存储器上的物理块，取0或1表示空间和占用。
  3. **空闲块链：**空的指向下一个空的指针
  4. **成组链接：**UNIX系统采用该方法，例如100个一组，每组1个空的登记下一组空闲的物理盘块。

- **文件共享和保护：**

  文件共享，不同的用户使用同一个文件。

  1. **硬链接：**两个文件目录指向同一个索引结点链接。
  2. **符号链接**：符号建立新的文件和目录，并于原来的文件建立路径映射。

- **文件保护（重点）：**

  1. **存取控制矩阵：**按人划分权限，但太大。
  2. **存储控制表：**按用户分类划分权限
  3. **用户权限表：**改进存储控制矩阵，将用户可存储的文件集中起来存入表中。
  4. **密码：**建立密码，对内容进行加密。

### 2.5、中间与构件

- **中间件（重点）：**操作系统和应用程序之间的被称之为中间件

- **构件（重点）：**又是组件，自包容、可复用的程序集，构件是独立是自包容的：

  （**设计构件组装->建立构件库->构件应用软件->测试与发布**）

  **构件组件模型优点如下：**构件的自包容性让系统拓展更加容易，设计良好的构件更容易被重用，降低软件的开发成本，架构颗粒度对整个系统更小，因此开发任务更加灵活。

  **缺点：**需要经验丰富的架构师

- **构件标准基本规范（重点）：**

  1. **公共对象请求代理架构（CORBA）：**三层次（对象请求代理（ORB）、公共对象服务和公共设施），软总线由ORB实现，在ORB 上定义了很多公共服务和规则，最上层定义了公共设施定义了构件框架。
  2. **CORBA CCM：**构件模型是OMG组织制定的一个用于开发和配置分布式应用服务器的构件模型规范，它主要包括如下内容：
     - **抽象构件模型：**服务端结构和构件间操作。
     - **构件容器结构：**用以提供通用的构件运行和管理环境，并支持安全、实物、持久状态等系统服务的集成。
     - **构件配置和打包规范：**CCM 使用打包技术来实现管理构件的二进制、多语言版本

### 2.6、嵌入式系统（重点）

- **组成：**嵌入式处理器、支撑硬件、嵌入式操作系统、支撑软件、应用软件

- **特点：**
  
  1. 专用性强
  2. 技术融合
  3. 软硬一体以软件为主
  4. 计算机资源少、考虑经济性
  5. 程序代码固化在非易失存储器中，提高了执行速度和系统可靠性
  6. 需要专门的开发工具和环境
  7. 体积小，价格低、工艺先进、性价比高、系统配置要求低
  8. 安全性的可靠性要求高
  
- **分类：**
  1. **嵌入式实时系统：**强实时（制定时间）、弱实时（大概时间）、在制定的时间内完成系统和外内部，同步或异步做出响应的系统。
  2. **嵌入式非实时系统：**安全攸关系统(不正确的功能会失效造成人员伤亡或者财产损失)、非安全攸关系统
  
- **软件组成：**
  
  1. **硬件层：**嵌入式系统提供运行支撑硬件环境，其核心是微处理器，存储器，IO接口和通用的设备以及总线、电源、时钟。
  2. **抽象层：**在硬件和软件层之间为抽象层，主要对硬件层进行抽象，(HAL)，提供虚拟的硬件资源，板级支持包（BSP）是一种硬件驱动软件，它是面向硬件层的硬件芯片或电路进行驱动。
  3. **操作系统层：**操作系统层主要由嵌入式操作系统，文件系统，图像用户接口，网络系统和通用组件等可配置模块组成。
  4. **应用层：**各种各样应用
  
- **嵌入式系统特点（重点）：**
  1. **可裁剪性：**根据系统功能需求，通过工具进行实用性功能加减，删除系统中不需要的模块，
  2. **可配置性：**根据系统运行时的功能进行可配置的能力，使嵌入式软件能根据系统的不同状态，不同容量，不同流程，进行能力拓展。
  3. **强实时：**嵌入式系统大多数都属于强实时系统，在规定时间内限时完成处理。
  4. **安全性：**安全性在指定系统在规定的条件下的时间内不发生事故的能力。
  5. **可靠性：**在特定条件下和规定的时间周期内程序执行所要求的功能和能力。
  6. **高确定性：**预先设计规划好，其行为不能随时间，状态的变迁而变化。
  
- **安全攸关系统（重点）：**

  1. **定义：**系统中，不可接受风险的软件，在航天航空，轨道交通，核工邻域等等。
  2. **目标：**
     - A级： 灾难性 ，无法 
     - B级： 危害性，严重减低
     - C级： 严重的，显著降低
     - D级： 不严重的，轻微降低
     - E级： 无影响，不会影响
  3. **过程：**软件开发过程、软件计划过程，软件综合过程。
  4. **数据：**软件生命周期文档、代码、报表、记录等所有产品


### 2.7、计算机网络

- **性能指标：**

1. **速率：**网速bit
  2. **带宽：频率宽带**
  3. **吞吐量：表示单位通过某个网络，吞吐受网络的带宽额度速率影响**
  4. **时延：**数据（报文、分组）从网络端到端的所需时间
     - **发送**时延：**第一个比特起算，到最后一个比特所发送所需时间。
     - **传播**时延：**信号在介质传播一定距离所花费的时间
     - **处理**时延：**检查、决定将分组导向何处的所需时间。
     - **排队**时延：**队列中，分组等待传输时间。
  5. **往返时间：**RTT，重要的网络性能指标，发送数据开始，到接收方确认总经历的时间。
  6. **利用率：**（信道利用率、网络利用率）两种，百分百表示。空闲信道利用率为零，网络利用率，信道利用率的平均值。

- **非性能指标**
  1. 费用、质量、标准化、可靠性、可拓展性、易管理和维护性
- **信道：**信源编码、信道编码、交织、脉冲形成和调制。相反地，收
  1. **信源编码：**** 模拟信号转化成模数转换，在进行压缩编码，最后形成数字信号（GSM\PCM）脉冲编码调制，模拟信号转换成二进制数字码流，利用RPE-LPT进行压缩。
  2. **信道编码：**增加冗余信息便于在接收端进行检错和纠错，剞劂信道，噪声和干扰导致的错误问题。
  3. **交织：**为解决信道误码导致的信道译码出错问题，通过交织将信道编码之后
  4. **脉冲形成：**减少带宽需求，将发送的数据转化成合适的波形，这就是脉冲形成。
  5. **调制：**调制是将信息承载到满足的信号要求是高载波信号过程。
- **复用技术：** 一条信道同时传输多路数据的技术
- **多址技术：**传输多个用户数据的技术。
- **网络拓扑：**
  
  1. **星型：**控制中心，构建简单，建网容易
  2. **树形：**成本低，扩充方便，灵活，方便查找链路路径
  3. **总线：**总线的负载能力是有限度的
  4. **环形结构: **首尾相连 ，闭合环单向流动，
  5. **网状结构：**任何彼此之间存在一条通信链路，任何结点故障不会影响其他节点之间的通信。
- **以太网技术：**计算机局域网组网技术，IEEE802.3 标出了以太网的技术标准。
1. **结构：**最大1518字节 ，最小64字节。
  2. **无线局域网：**微波、激光、红外线灯无线电波作为传输介质。
  3. **广域网：**通信子网、资源子网
  4. **城域网：** DQDB ，双总线构成，所有计算机都连接在上面
  5. **第五代移动通信(5G):** 高速率、低延迟、大链接、网络切片和服务化架构

### 4.2、信息加密

- **对称密钥加密算法：**加密 解密，优点速度快，适合明文加密 。
- **缺点：**加密强度不高，且密钥分发困男需要安全可靠的途径
- **实现：**大量数据的加密
- **常见算法：**
  1. **DES：**56位密钥、64数据库 速度快，密钥易于产生
  2. **3DES：**密钥长度112位、用2个56位的密钥K1、K2
  3. **IDEA：**使用128位的密钥
- **非对称加密算法：**公钥 和 私钥 的区别RAS
- **优点：**密钥分发简单
- **缺点：**速度慢
- **数字摘要算法：**杂凑算法，主要用于数字签名，进行比较，单向性、抗弱碰撞性
  1. **消息摘要算法：**用于确保信息传输完整的一致性
  2. **安全散列算法：**MD5和SHA的散列值分别是128 位160位 SM3(国密算法)256位

### 4.3、密钥管理技术

- **分配与管理：**自动分配密钥机制、提高系统效率、二是减少系统中驻留的密钥量
- **密钥分配方式：**
  1. 物理手段发送给另一方
  2. 第三方选取密钥，通过物理手段发送给A和B 
  3. AB都有一个密钥其中一方选取密钥后，用已有的密钥加密该新密钥发送给另一方
  4. 三方各有一道保密信道，某方选取密钥后，保密信道自动发送。
- **非对称加密密钥管理技术：**
  1. **公开发布：**将密钥发送给每一个其他用户，缺点任何人都可以伪造密钥公开发布。
  2. **公用目录表：**公钥动态目标，建立条目含有用户名和公开密钥的两个数据项，每个用户都可以注册自己的公钥，缺点是：攻击者如果盗用，则可以伪造一个目录表。
  3. **公钥管理机构：**与目录表类似、采取更加严密的措施来增加安全性。用户都知道管理机构的公钥，缺点每一位都必须求助于管理机构
  4. **公钥证书：**公钥帧数管理机构为用户建立的，公钥及用户的身份和时间戳等，所有的数据项CA用自己的私钥签名形成证书。

### 4.4、访问控制及数字签名技术

- **要素：**主体（用户本身）、客体（信息、资源、对象）、控制策略（操作约束，认证，审计）
- **实现技术：**
  1. **访问控制矩阵：**表、可读可写，三要素
  2. **访问控制表：** 按列，每个客体有一个访问控制表
  3. **能力表：** 行，以主体为索引，保存访问矩阵，对于主体能访问的能力信息
  4. **授权关系表：**主体排序得到能力表的效率、按客体排序得到控制表的效率
- **数字签名：**
  - **概括：**单项函数和要传输的报文进行处理，解决否认、伪造、篡改和冒充的问题
  - **条件：**可信、不可伪造、不可重用、不可改变、不可抵赖

### 4.5、信息安全抗攻击技术

- **密钥选择：**数据加密（DK）、密钥加密（KK）

- **因素：**增大密钥空间、选择强钥、密钥的随机性

- **拒绝服务攻击与防御（模式）：**

  1. **消耗资源：**（拒绝服务攻击、消耗磁盘空间、消耗CPU资源和内存资源）
  2. **破坏或更改配置信息：**攻击配置文件，向外提供服务。
  3. **物理破坏或更改网络部件：**计算器、路由器、网络配线室
  4. **利用服务程序中的错误使服务失效**

- **分布式拒绝服务攻击：**

  - **结构：**客户端（运行攻击者主机上）、主控端（运行已被入侵和获得控制的主机上）、代理端（负责接受命令和实施攻击）

- **拒绝服务攻击防御方法：**

  1. 加强对数据包的特征识别
  2. 防火墙
  3. 对通信数据量进行统计
  4. 尽可能修复发现问题和漏洞

- **欺骗攻击：**

  1. **ARP欺骗：**（以太网地址解析协议）IP地址转化为MAC地址的，通过欺骗局域网访问者的网关MAC地址，使网络不通

     **措施：**固化ARP表、ARP服务器、双向绑定、ARP防护软件

  2. **DNS欺骗：**冒充服务器，冒名顶替

     **措施：**被动监听检测、虚假报文探测、交叉检测查询

  3. **IP欺骗：**源地址经过修改的IP数据包，隐藏发送方的身份，入口过滤是防范欺骗的一种常见方式

  4. **端口扫描：**与目标主机端口建立链接

  5. **强化TCP IP 堆栈低于拒绝服务攻击：**同步风暴、ICMP 攻击（不超过64kb）、SNMP 攻击(网络协议)接管网络。

### 4.6、计算机安全保护等级

- **级别：**自审标结验
  1. **用户自主保护级：**隔离用户和数据，使用户具有自主保护的能力
  2. **系统审计保护等级：**通过登录规程、审计安全、相关时间和隔离资源使用户对自己的行为负责。
  3. **安全标记保护等级：**提供相关安全策略模型、对主体和客体强制访问进行非形式化的描述。
  4. **结构化保护：**将第三季系统中的自主和强制访问控制拓展到所有主体和客体+考虑隐蔽通道+加强了鉴别机制，增加配置管理控制
  5. **访问验证保护级：**访问监控器仲裁主体对客体的全部访问，访问监控器抗篡改，必须足够小，能分析和测试，降低复杂性最小程度，扩充审计机制，提供系统恢复机制。
- **风险评估的实施流程：**
  1. 评估范围、评估的目标、组织结构、风险评估方法，最高管理者对风险评估策划的批准
  2. **风险评估要素：**脆弱性、资产、风险、安全措施。

  3. **设计原则（重点）：**高内聚、低耦合、内聚表示各代码模块之间的紧密程度
     - **内聚：**表示紧密程度
     - **高内聚：**功顺通过时逻偶
     - **耦合：**模块之间联系的程度，少一些联系
       1. **非直接耦合：**两个模块之间没有直接关系，通过上级模块的控制和调用来实现的
       2. **数据耦合：**模块借助参数来表达简单数据
       3. **标记耦合：**模块通过参数来表达传递记录等复杂信息
       4. **控制耦合：**控制模块内部逻辑
       5. **通信耦合：**一组模块公用一组输入信息、共享输入或输出信息
       6. **公共耦合：**多个模块之间访问同一一个公共数据环境，全局数据环境
       7. **内容耦合：**直接访问另外一个模块的内部数据，表示关联程度最高

### 5.4、软件测试

- **语句：**语句覆盖、判定覆盖、条件覆盖、条件判定覆盖、条件组合覆盖、路径覆盖。

  1. **语句：**确保每条语句至少执行一次
  2. **判断：**两种结果，至少都出现一次
  3. **条件：**每个条件的两种结构都至少出现一次
  4. **路径：**执行路径至少执行一次

  路径覆盖包含所有语句的执行，因此路径可以代替其他语句

- **动态测试：**

  1. **白盒：**结构测试、透明盒测试、逻辑驱动测试或者基于代码的测试，不许了解程序内部的结构和处理过程。（基本路径测试、循环覆盖、逻辑覆盖）单元测试阶段
  2. **黑盒：**不考虑程序内部构件和处理过程（等价类划分、边界值分析、错误推测、因果图）软件测试阶段
  3. **灰盒：**白盒测试与黑盒测试之间的测试内容

- **阶段：**

  1. **单元测试：**最小单位模块，由开发人员自己负责，借助于驱动模块和桩模块，在详细设计阶段完成的。白盒

  2. **集成测试：**组装和联合测试，测试模块之间的协作性，一次组装和增量组装，软件概要设计阶段完成吗，白盒黑盒一起用

     - **驱动模块：**被测试的主程序
     - **桩模块：**用于代替被测试调用的子模块

     自顶向下组装不需要驱动模块，自顶向上，不需要桩模块。冒烟测试称之为版本提交和提交测试

  3. **确认测试：**有效性测试，软件功能和性能是否满足用户需求。

     - **内部确认：**由软件开发组装内部按软件需求说明书进行测试。
     - **Alpha测试：**开发环境进行测试
     - **Beta测试：**用户在实际（生产）环境进行测试
     - **验收测试：**自己提的需求，在交互前以用户为主进行测试

  4. **系统测试：**连通性、确认软件、硬件、外设、网络等元素结合在一起；在需求分析阶段完成，黑盒

  5. **压力测试：**瓶颈的最大性能点

  6. **负载测试：**超负荷负载运行中程序是否能承担

  7. **强度测试：**限度系统资源特别低的情况下考察软件运行情况

  8. **容量测试：**在线的最大用户数

  9. **其他测试：**

     1. **AB测试：**两个组进行测试，取优
     2. **Web测试：**web应用类测试
     3. **链接测试：**页面切换
     4. **表单测试：**提交信息时希望表单能正常工作
     5. **自动化测试：**软件测试自动化

  10. **回归测试：**合格性测试、用来验证软件于用户需求的一致性

### 5.6、基于构件的软件工程CBSE

- **思想：**分布式对象技术，购买尔不是重新构造的哲学，将软件开发的重点从程序编写转移到基于已有构件的组装，以便于更块的构造系统。
- **特征（重点）：**
  1. **可组装：**接口进行
  2. **可部署：**自包含，二进制的
  3. **文档化：**完全文档化
  4. **独立性：**构件是独立的，可以无其他特殊构件的情况下进行组装和部署，构件需要显示声明对其他构件的依赖
  5. **标准化：**符合某种标准化的构件模型
- **过程（重点）：**
  1. 系统需求概览
  2. 识别候选构建
  3. 发现构件修改需求
  4. 体系结构设计
  5. 定制和适配
  6. 组装构件创建系统
- **构件组装（重点）：**构件相互之间集成和胶水代码的概念，创造一个系统或者一个构件的过程
  1. **顺序组装：**按顺序调用以已存在的构件，用两个已经存在的构件来创造一个新的构件
  2. **层次组装：**直接调用一个构件所提供的服务时，兼容直接调用否则用胶水代码实现转换
  3. **叠加组装：**2个或2个以上的构件来创建一个新的构件的时候，不相互依赖也不相互调用

### 5.7、项目管理

- **思想：**人员、产品、过程和项目的分析管理活动

- **工作分解的基本要求：**
  1. WBS工作包可控可管理的，不能过于复杂
  2. 任务分解不能太细，不超过6层
  3. 每个工作都要有一个可交付成果
  4. 每个任务必须要有哦明确的标准
  5. WBS必须有利于责任分配
- **任务活动图：**前导图（单代号），箭线图法（双代号网络法）
- **软件配置管理SCM：** 标识、组织、控制修改技术、使错误降为最小并最有效提高生产效率

- **配置库：** 
  1. **动态库：**（开发库、程序员库、工作库），随意修改
  2. **受控库：**主库和系统，用例当前基线和管理基线的变更
  3. **静态库：**软件产品库，存档于各种各样的广泛使用的已经发布的基线，不能修改
